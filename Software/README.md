```java
69. 소프트웨어 생명주기 모형 중 Spiral Model에 대한 설명으로 가장 옳지 않은 것은? 4번
① 대규모 시스템에 적합하다.
② 개발 순서는 계획 및 정의, 위험 분석, 공학적 개발, 고객 평가 순으로 진행된다.
③ 소프트웨어를 개발하면서 발생할 수 있는 위험을 관리하 고 최소화하는 것을 목적으로 한다.
④ 개발 과정의 앞 단계가 완료되어야만 다음 단계로 넘어 갈 수 있는 선형 순차적 모형이다.


70. 소프트웨어의 품질 목표 중에서 옳고 일관된 결과를 얻기 위 하여 요구된 기능을 수행할 수 있는 정도를 나타내는 것은? 2번

② 신뢰성(Reliability)
③ 효율성(Efficiency)
④ 무결성(Integrity)

78. Alien Code에 대한 설명으로 가장 옳은 것은? 2번
① 프로그램의 로직이 복잡하여 이해하기 어려운 프로그램 을 의미한다.
② 아주 오래되거나 참고 문서 또는 개발자가 없어 유지보수 작업이 어려운 프로그램을 의미한다.
③ 오류(Error)가 없어 디버깅 과정이 필요 없는 프로그램을 의미한다.
④ 차세대 언어를 사용해 인공지능적인 API를 제공함으로써 사용자가 직접 작성한 프로그램을 의미한다.

익스트림 프로그래밍(XP)
∙ 켄트 백 등이 제안한 소프트웨어 개발 방법
∙ 의사소통, 단순성, 피드백, 테스트, 자신감 강조
∙ 애자일 개발 프로세스의 대표자로 애자일 개발 프로세스의 보급에 큰 역할
∙ 고객과 함께 2주 정도의 반복 개발을 하고, 테스트와 우선 개발을 특징으로 하는 명시적인 기술
∙ 전통적인 소프트웨어 개발 방법론과는 달리 문서를 강조하지 않고 변경을 장려
∙ 개발 중인 시스템을 최대한 빨리 고객한테 보여줌으로써 고객들이 원하는 변경 사항을 빨리 도출
∙ 수시로 발생하는 고객의 요구 사항에 대처하고, 고객이 원하는 SW를 고객이 원하는 시간에 인도하기 위해서는 고객과 팀원 간의 대화를 통해 해결


스크럼(SCRUM)
∙ 프로젝트를 30일마다 동작 가능한 제품을 제공하는 스플린트로 분리
∙ 팀은 매일 scrum(15분 정도) meeting
 
통합 프로세스 모델 UP(Unified process)
∙ 객체지향적으로 소프트웨어를 개발하는 프로세스를 정의한 방법론
∙ 소프트웨어를 개발하기 위한 일련의 활동과 작업, 그리고 산출물들을 작성하는 방법을 제시
∙ 야콥슨의 유스케이스 중심적(Use-case driven) 방법을 사용
∙ 소프트웨어 개발 단계를 도입, 상세(구체화, 정련), 구축, 이행(전이)의 프로젝트 단계로 나누고 해당 단계를 여러 번 반복하면서 실행
∙ 각 단계는 요구사항 도출, 분석, 설계, 구현, 평가의 개발 생명주기가 포함되어 있음
∙ 반복적(Iterative)이고, 점진적(Incremental)으로 개발
∙ 유스케이스(Usecase)를 기반으로 함 : 요구사항을 식별하고, 정의하는데 있어서 UML의 유스케이스 사용
∙ 아키텍쳐(Architecture) 중심의 개발을 지향
∙ 위험 관리를 중시
```
