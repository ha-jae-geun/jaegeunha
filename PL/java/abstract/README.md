# 추상클래스와 인터페이스 차이
```java
이러한 상황일 때는 추상클래스를 고려해야 합니다:

코드를 다른 여러 개의 관련 있는 클래스들과 공유해야 할 때.
공통적인 메서드, 필드 혹은 public이 아닌 접근제어자로 한 개의 클래스를 여러 클래스가 상속할 때.
static, final이 아닌 필드를 생성하고 싶을 때. 추상클래스는 객체에 접근하고 수정할 수 있게 합니다.


이러한 상활일 때는 인터페이스를 고려해야 합니다.

인터페이스와 연관이 없는 클래스들이 구현하려고 할 때.
특정 데이터 타입의 행동을 만들고 싶을 때. 하지만 이러한 행동을 누가 구현하는지 걱정하지 않아도 될 때.
여러 타입으로 상속받을 때 이득을 받고 싶을 때.
```
