# 에러 검출
```java
네트워크의 전송 데이터 오류 검출에 대한 설명으로 옳지 않은 것은? 4번
① 체크섬(checksum)은 1의 보수 방법을 사용한다.
② 순환중복검사(CRC)는 모듈로-2 연산을 주로 사용한다.
③ 전송할 데이터에 대한 중복 정보를 활용하여 오류를 검출한다
④ 단일 패리티 비트를 사용하는 패리티 검사는 홀수 개의 비트에 오류가 발생하면 오류를 발견할 수 없다.

∙ 패리티 검사는 홀수 개의 비트의 오류는 검출이 가능하며 짝수 개의 비트에 오류가 발생하였을 때 검출이 되지 않습니다.


데이터 전송 중에 발생하는 에러를 검출하는 방식으로 옳지 않은 것은?
① 패리티(parity) 검사 방식
② 검사합(checksum) 방식
③ CRC 방식
④ BCD 부호 방식

BCD 코드(Binary Coded Decimal, 2진화 10진 코드)는 10진수 1자리의 수를 2진수 4Bit로 표현하는 방식입니다. 
4Bit의 2진수 각 Bit가 8(2³), 4(2²), 2(2¹), 1(2~~[SP]0~~[/SP])의 자리값을 가지므로 8421코드라고도 하며 
대표적인 가중치 코드 (Weight Code)입니다. BCD 코드는 에러 검출 기능이 없습니다.
```
