한 번 생성되면 그 인증값이 임시적으로 한 번만 유효한 비밀번

1 S/KEY 방식
해시 체인에 기반하여 OTP를 생성하는 방식이다.
클라이언트에서 정한 임의의 비밀키를 서버로 전송하여, 해시 알
고리즘을 통해 해시값을 구하는 작업을 n번 반복하고, 그렇게 생

4 Challenge-Response 방식

에 전송하면, 클라이언트가 그 값으로 OTP를 생성해 응답한 값
으로 인증하는 방식이다.
